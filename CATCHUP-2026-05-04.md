## What This Project Is

CSD Post Engine at `C:\Users\Dannielle\ClaudeOS\csd-engine\`

Node.js engine that generates daily social media posts for the Instagram account @choosingsobrietydaily.
Run with: `node run.js morning / midday / night`

It:
1. Pulls a quote, theme, visual style, and context/fact from Airtable pools
2. Builds an image prompt + WhatsApp caption + IG caption
3. Generates the image via `gemini --yolo "/generate ..."` (Nano Banana MCP)
4. Writes a record to Airtable "Daily Posts" table
5. Archives the image to `G:\My Drive\Choosing Sobriety Daily\CSD Post v4`

## What Was Done Today

### 1. Fixed 4 recurring run.js issues (linter keeps reverting these — reapply each session)

- `automation_status` on create: `'Ready'` → `'Draft'` (Airtable valid option)
- `status` on create: `'Draft'` → `'Todo'` (Airtable only accepts Todo/In progress/Done)
- `markCompleted()`: replaced `fetch()` with `https.request()` — fetch fails after `execSync` in Node.js
- Image generation command: `nano-banana /generate "..."` → `gemini --yolo "/generate ..."` 
- Removed `!fs.existsSync(imagePath)` guard so same-day re-runs can overwrite

### 2. Fixed context rotation bug (pull.js)

**Problem:** Every post was showing the same "Did You Know" fact — "Real growth takes root quietly before it shows itself clearly." — because `pickContext` selects the oldest Last Used record but never updated it after picking.

**Fix:** Added `updateContextUsed(recordId, currentCount)` in `pull.js` — PATCHes the Airtable context record with today's date and increments Usage Count immediately after picking. Each post now rotates to a different fact.

### 3. Logo overlay (run.js + prompt-builder.js)

**Problem:** AI was rendering the CSD logo inconsistently.

**Fix:**
- Removed logo instructions from `prompt-builder.js` — AI now leaves the top-right corner empty
- After image generates, `sharp` (Node.js library) precisely overlays `csd-logo.png` onto the top-right corner at 14% of image width, 2.5% padding
- Logo composite happens before Drive archive, so the archived file already has the logo

Logo size on a 1024×1024 image: 143px, positioned 26px from top-right corner.
**User has not yet confirmed logo placement looks right — this may need size/position tweaks.**
